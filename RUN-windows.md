# Как запустить проект — Windows (для фронтенда)

Нужно **два процесса**: бэкенд (API) и статический сервер для фронта.
Держи их в **двух отдельных окнах** PowerShell.

Требования: установленный **Node.js 22+** (проверь: `node --version`).
Node с сайта nodejs.org ставится вместе с `npm` — отдельно ничего не нужно.

---

## Разовая подготовка (только первый раз)

Открой PowerShell в папке проекта:
```powershell
cd back
copy .env.example .env      # конфиг: JWT_SECRET и PORT=3000
npm install                 # поставить зависимости
```

> `.env` в `.gitignore`, поэтому в репозитории его нет — создаём из шаблона командой `copy`.
> Если файл `back\.env` уже есть — шаг пропускаем.

---

## Окно 1 — Бэкенд (API)

```powershell
cd back
npm start
```
Увидишь:
```
БД инициализирована: ...\back\data\plant_helper.db
Plant Helper API запущен: http://localhost:3000
```
Оставь окно открытым. Остановить — `Ctrl+C`.

**Проверка:** открой в браузере http://localhost:3000/api/health → `{"ok":true}`

---

## Окно 2 — Клиент (фронт)

Фронт — статические файлы, их нужно отдавать по http (не открывать `index.html` двойным кликом — часть браузеров режет запросы с `file://`).

Выбери **любой** из способов:

**А) VS Code + расширение Live Server (проще всего)**
1. Установить расширение «Live Server» (Ritwick Dey).
2. Открыть папку `front` в VS Code.
3. ПКМ по `index.html` → **Open with Live Server**.
   Обычно откроется на `http://127.0.0.1:5500`.

**Б) Через Node (если Live Server нет)**
```powershell
cd front
npx serve -l 5500
```
(`npx` скачает `serve` при первом запуске — нужен интернет.)

**В) Через Python (если установлен)**
```powershell
cd front
python -m http.server 5500
```

Оставь окно/вкладку открытым.

---

## Открыть приложение

Браузер → **http://localhost:5500** (или адрес, который показал Live Server).

Порядок: **сначала** бэкенд (Окно 1), **потом** фронт (Окно 2).

---

## Важно про данные

Часть экранов сейчас работает на **моках** (данные зашиты в `front/js/api.js`) —
это временно, пока бэкенд доделывает эндпоинты. Реально с сервера уже идёт только
справочник (`/api/plants`). По мере готовности бэка переключаем флаги
`MOCKS_ENABLED` в `api.js` с `true` на `false` — код при этом не меняется.

---

## Остановить всё

В каждом окне — `Ctrl+C`. Для Live Server — кнопка «Port: 5500» внизу VS Code → клик.

---

## Если что-то не так

| Симптом | Решение |
|---|---|
| `node : команда не найдена` | установить Node.js 22+ с nodejs.org, перезапустить PowerShell |
| `EADDRINUSE :3000` | порт занят: закрой старое окно с сервером или поменяй `PORT` в `back\.env` |
| фронт не грузит справочник | бэк не запущен — проверь Окно 1 и http://localhost:3000/api/health |
| `npm install` ругается | проверь, что ты в папке `back`, где лежит `package.json` |
