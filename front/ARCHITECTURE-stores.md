# Сторы — единый источник правды (важно прочитать)

Мы вынесли доменные данные из вьюмоделей в **сторы** — по одному объекту-владельцу
на домен, один экземпляр на всё приложение (создаётся в `ServiceContainer`, инжектится
во вьюмодели). Это убирает класс багов «протухший стейт» (когда в одном экране данные
изменились, а в другом остались старые).

## Что появилось
```
stores/CollectionStore.js  — сад пользователя (единственная копия + мутации)
stores/FavoritesStore.js   — избранное (множество id)
stores/RemindersStore.js   — напоминания (ПРОИЗВОДНОЕ: сам перечитывается при смене коллекции)
```

## Главное правило
**ViewModel НЕ хранит свою копию доменных данных и НЕ ходит в сервис напрямую.**
Она:
1. читает данные из стора (`store.items` / `store.ids`),
2. подписывается на `store.on("change", ...)` и пересчитывает своё представление,
3. мутации делает через методы стора (`collectionStore.markWatered(id)` и т.п.).

```js
// в конструкторе VM
this.collectionStore.on("change", (items) => { this.state = {...this.state, items}; this.emit("change", this.state); });

// действие пользователя
markWatered(id) { this.collectionStore.markWatered(id); } // стор сам разошлёт "change" всем
```

## Почему это чинит баги
Полил растение из «Задач» → `collectionStore.markWatered` → стор перечитал → разослал
`change` → «Мой сад», «Главная», «Задачи» обновились одновременно. **Кнопок «полить»
может быть сколько угодно (сад, задачи, главная) — все идут через один метод стора,
рассинхрон невозможен by design.** Источник правды — данные в сторе, а не один UI.

## Что НЕ делать
- ❌ не кэшируй коллекцию/избранное локально в новой вьюмодели — бери из стора;
- ❌ не вызывай `collectionService.markWatered` напрямую из VM — только через `collectionStore`;
- ❌ не добавляй ручные `xxxViewModel.load()` в `app.js` при логине — грузятся сторы,
  экраны подписаны и обновятся сами (см. обработчик auth в `app.js`).

## Добавляешь новый экран, работающий с этими данными?
Инжектни нужный стор из контейнера, подпишись на `change`, читай `store.items`.
И всё — синхронизация с остальными экранами получится автоматически.
